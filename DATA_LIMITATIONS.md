# データ制約について — 実データ接続時に何が正確で、何が近似か

このドキュメントは「動いているように見えて、実は違うものを計算している」状態を防ぐために存在する。
36手法それぞれについて、J-Quants Light プラン + 無料の米国株データ source で、
どこまで正確に計算できるかを正直に記録する。

## 前提となる制約

| データ | 制約 |
|---|---|
| J-Quants Light: 株価 | 過去5年前まで(それ以前は取得不可) |
| J-Quants Light: 財務情報 | "サマリー"レベルのみ。貸借対照表・キャッシュフロー明細は Standard 以上 |
| J-Quants: 信用取引残高・空売り比率 | Standard 以上(本アプリでは該当手法を実装から除外済み) |
| 米国株: 現在の株価・PER・EPS・配当利回り・時価総額 | GAS + GOOGLEFINANCE() で取得可能 |
| 米国株: ROE・ROIC・FCF・粗利益率など詳細財務 | **GOOGLEFINANCE では取得不可**。無料での安定取得手段が現時点でない |
| 米国株: 日次株価ヒストリカル(5年超) | Stooq(無料CSV)で取得 |

## 日本株 — 手法別の精度

| 精度 | 手法 |
|---|---|
| **正確**(J-Quantsの生データをそのまま使用) | 12ヶ月モメンタム、52週高値接近、セクター相対力、パーフェクトオーダー、出来高ブレイク、押し目買い、ボラ収縮ブレイク、相対力ライン新高値、フォーカスリスト、低PER×高ROE、PBR分解、高配当×健全性、連続増収増益、低ベータ、ディフェンシブ配当、業績安定性、マルチファクター |
| **近似**(計算方法を簡略化。方向性は合っているが数値の精度は劣る) | FCF利回り(→ 営業CF利回りで代替。CapEx が取得不可のため)、EV/EBITDA(→ EV/EBIT相当。減価償却費が取得不可のため)、ROIC超過(→ 有利子負債を無視した近似。投下資本ではなく自己資本ベース)、マジックフォーミュラ(ROICが上記近似のため連動)、成長加速(四半期の累積値からの逆算。真の四半期値ではない)、決算ドリフト(同様) |
| **簡易版**(判定項目を減らした縮小版) | Piotroski Fスコア(9項目中、貸借対照表明細が必要な3項目を除いた6項目版。名称は維持しているが厳密な原典とは異なる) |
| **当面 0件表示**(必要なデータが取得不可。無理に近似せず、正直に「該当なし」を出す) | 粗利益率(真の売上原価が取得不可)、財務安全性(Altman Zスコア。負債明細が必要)、ネットネット(流動資産・流動負債の明細が必要)、自社株買い、株式分割 |

**自社株買い・株式分割**は、TDnet の適時開示文書を構造化データとして提供する無料APIが見当たらず、
J-Quants Light にも該当エンドポイントがない。実装は残しているが、当面はデータソースが無いため
常に0件になる。将来的にTDnet開示のスクレイピング(規約を確認の上)を追加する余地がある。

**増配・累進配当**は配当予想(ForecastDividendPerShareAnnual)の変化から検出しており、これは正確に計算できる。

## 米国株 — 手法別の精度

米国株は日本株よりデータが薄い。**GOOGLEFINANCE から得られるのは株価・PER・EPS・配当利回り・
時価総額・52週高値安値程度**で、ROE・ROIC・FCF・粗利益率などの詳細財務は無料では安定して
取得できていない。

| 精度 | 手法 |
|---|---|
| **正確** | 12ヶ月モメンタム、52週高値接近、セクター相対力、パーフェクトオーダー、出来高ブレイク、押し目買い、ボラ収縮ブレイク、相対力ライン新高値、フォーカスリスト、低ベータ |
| **一部正確**(PERは正確、ROEなど収益性は取得不可) | 低PER×高ROE、マジックフォーミュラ、PEGレシオ |
| **当面0件表示** | Piotroski Fスコア、ROE継続性、粗利益率、会計発生高、ROIC超過、財務安全性、CANSLIM(四半期EPS成長率が必要)、成長加速、FCF利回り、EV/EBITDA、高配当×健全性(配当性向にNI要)、ディフェンシブ配当、業績安定性、マルチバガー候補、マルチファクター(バリュー/クオリティ成分が欠落するため実質モメンタムのみになり、設計意図と異なる) |

**率直に言うと**、米国株のバリュー・クオリティ・グロース系手法を本気で機能させるには、
SEC EDGAR の Company Facts API(無料・公式・XBRL財務データ)を追加で組み込む必要がある。
今回のスコープには含めていない。次にやるべき作業として明記しておく。

## 権利確定日・決算発表日・株主優待について(推定データの正直な扱い)

| データ | 導出方法 | 精度 |
|---|---|---|
| 権利確定日 | 事業年度末(と期央=中間配当基準日)の月末を権利確定日とみなす | **近似**。大半の企業は期末=権利確定だが、厳密な「権利付き最終日」(2営業日前)とは数日ズレる。権利取り系手法の売買タイミングには数日のマージンを持たせてある |
| 決算発表日 | 前年の同時期の開示日+1年で推定 | **推定**。日本企業は毎年ほぼ同時期に発表する慣行があるため±1週間程度で当たるが、確定情報ではない。UI上も必ず「推定」と表示する |
| 株主優待の有無 | 無料の構造化データが**存在しない**。任意の手動リスト(batch/yutai-jp.json に銘柄コードの配列を置く)で補強 | リスト未整備でも権利取り系手法は配当利回り条件だけで動作する。優待情報を本気で使うなら、証券会社サイト等から自分の対象銘柄分だけ手動転記する運用 |

## 実運用を始める前にやるべきこと

1. 最初の数営業日は、この一覧と実際に0件になる手法を突き合わせ、想定通りかを確認する。
2. 「近似」に分類された手法のスコア閾値(`STRATEGIES` 内の score() 関数の数値)は、
   検証用の乱数データを基準に決めたものであり、実データの分布とは合っていない可能性が高い。
   数週間分のデータが溜まったら、閾値を実データの分布(パーセンタイル)に合わせて調整すること。
3. 米国株のバリュー・クオリティ系を本気で使いたい場合は、SEC EDGAR 連携を追加実装する。
